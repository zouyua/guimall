## 🎯 核心功能特性

### 1. 属性分类计数自动更新
- 创建属性时，自动更新分类的 `attributeCount` 或 `paramCount`
- 删除属性时，自动减少对应计数
- 修改属性分类或类型时，自动调整新旧分类的计数

### 2. 数据完整性校验
- 创建/修改时检查名称重复
- 删除时检查是否有关联数据
- 外键关联校验（属性分类是否存在）

### 3. 事务管理
- 使用 `@Transactional` 确保数据一致性
- 创建/修改/删除操作都在事务中执行

### 4. 灵活查询
- 支持按分类ID查询属性
- 支持按属性类型（规格/参数）筛选
- 支持属性名称模糊搜索
- 支持排序（按sort字段和ID）

## 📊 数据库表结构

### pms_product_attribute_category（商品属性分类表）
```sql
CREATE TABLE `pms_product_attribute_category` (
    `id` bigint NOT NULL AUTO_INCREMENT,
    `name` varchar(64) DEFAULT NULL COMMENT '属性分类名称',
    `attribute_count` int DEFAULT '0' COMMENT '属性数量',
    `param_count` int DEFAULT '0' COMMENT '参数数量',
    PRIMARY KEY (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='商品属性分类';
```

### pms_product_attribute（商品属性表）
```sql
CREATE TABLE `pms_product_attribute` (
    `id` bigint NOT NULL AUTO_INCREMENT,
    `product_attribute_category_id` bigint DEFAULT NULL COMMENT '属性分类ID',
    `name` varchar(64) DEFAULT NULL COMMENT '属性名称',
    `select_type` int DEFAULT '0' COMMENT '选择类型:0唯一1单选2多选',
    `input_type` int DEFAULT '0' COMMENT '输入类型:0手动1列表',
    `input_list` varchar(255) DEFAULT NULL COMMENT '可选值列表',
    `sort` int DEFAULT '0' COMMENT '排序',
    `filter_type` int DEFAULT '0' COMMENT '筛选类型',
    `search_type` int DEFAULT '0' COMMENT '搜索类型',
    `related_status` int DEFAULT '0' COMMENT '是否关联；0->不关联；1->关联',
    `hand_add_status` int DEFAULT '0' COMMENT '是否支持手动新增',
    `type` int DEFAULT '0' COMMENT '属性的类型；0规格1参数',
    PRIMARY KEY (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='商品属性';
```

### pms_product_attribute_value（商品属性值表）
```sql
CREATE TABLE `pms_product_attribute_value` (
    `id` bigint NOT NULL AUTO_INCREMENT,
    `product_id` bigint DEFAULT NULL,
    `product_attribute_id` bigint DEFAULT NULL,
    `value` varchar(255) DEFAULT NULL COMMENT '属性值',
    PRIMARY KEY (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='商品属性值';
```
